Quick Facts

Brain dump this at the start of the exam. Key facts to memorize cold.

Which Shortest Path Algorithm?
QuestionAnswer
All weights >= 0, one source?Dijkstra's
Negative weights possible, one source?Bellman-Ford
Need all pairs?Floyd-Warshall
Unweighted graph (hops only)?BFS
Negative cycle from one source?Bellman-Ford (iter[n] vs iter[n-1])
Negative cycle anywhere in graph?Floyd-Warshall (dist[v][v] < 0)
Any cycle at all (not negative)?DFS (back edge = cycle)
What Each Output Variable Means
VariableAlgorithm(s)What It Is
dist[]BFS, Dijkstra's, Bellman-FordShortest distance from source s to every vertex. One number per vertex.
dist[][]Floyd-WarshallShortest distance from every vertex to every other vertex. dist[u][v] = shortest u to v.
prev[]BFS, DFS, Dijkstra's, Bellman-Ford, Prim'sParent pointer. Follow prev[] backwards from t to s to reconstruct the path. O(n).
ccnum[]DFS, SCCConnected component number. Vertices with same ccnum are in the same component.
pre[]DFSPre-visit timestamp. When DFS first discovered the vertex.
post[]DFSPost-visit timestamp. When DFS finished exploring the vertex. THE IMPORTANT ONE.
order[]Topo SortVertices listed so all edges go left to right. Sources first, sinks last.
G_SCCSCCThe metagraph. A new, smaller adjacency list. Always a DAG.
edges[]Kruskal'sList of n-1 edges in the MST. Must convert to adjacency list before BFS/DFS.
iter[i][v]Bellman-FordShortest distance from s to v using at most i edges. dist[] = iter[n-1].
iter[k][u][v]Floyd-WarshallShortest distance from u to v using only vertices {1..k} as intermediates. dist[][] = iter[n].
flow[], CFord-Fulkerson, Edmonds-KarpFlow on each edge, and C = total max flow value.
assignments[]2-SATTrue/false value for each variable, or "no solution."
Cycle Detection
TypeHowDetails
Any cycle (directed)DFSBack edge exists iff post[u] < post[v] for edge u->v
Negative cycle from source sBellman-FordIf iter[n] differs from iter[n-1], negative cycle reachable from s
Negative cycle anywhereFloyd-WarshallIf any dist[v][v] < 0, negative cycle exists through v
No cycle (verify DAG)DFSNo back edges found = DAG. Safe to run Topo Sort.

Topo Sort Facts
FactDetail
Only works on DAGsCheck for cycles with DFS first if unsure
order[] = decreasing post orderHighest post number first
Sources first, sinks lastAll edges go left to right in the ordering
Process backwards for DPSinks have no neighbors, so start there and work toward sources
Unique topo sort iff Hamiltonian pathPath touching all vertices exists iff only one valid ordering
Max Flow Key Facts
FactDetail
Max-Flow = Min-CutThe maximum flow always equals the minimum cut capacity
No flow can exceed any cutFor any flow f and any cut (L,R): size(f) ≤ cap(L,R)
Min cut = split vertices into L and Rs in L, t in R. Cut capacity = sum of edges from L to R.
Residual graph = original graphMeans flow is zero (no flow sent on any edge)
No s-t path in residual graphCurrent flow is maximum (algorithm terminates)
Ford-Fulkerson: O(mC)C = max flow value. Requires integer capacities. Can be slow.
Edmonds-Karp: O(nm²)Uses BFS for augmenting paths. Works with any positive capacities.
Augmenting along a pathSend flow = min capacity along the path. At least one edge saturates per round.
Spanning Tree Facts
FactDetail
MST includes all n verticesExactly n-1 edges, connected, no cycles
Cut propertyLightest edge across any cut must be in MST
Every MST edge is lightest across some cutRemove it, tree splits into two groups = that's the cut
Distinct weights = unique MSTNo ties means only one possible MST
Max spanning treeNegate all weights, run Kruskal's
Bottleneck / widest pathPath from s to t in MST minimizes the heaviest edge
Kruskal's output = edges[]Must convert to adjacency list before BFS/DFS
Prim's output = prev[]Parent pointers, like Dijkstra's output

Common Traps
TrapWhy It's Wrong
Dijkstra's with negative weightsDijkstra's FAILS. Use Bellman-Ford instead.
ccnum[] on directed graph (from DFS)Useless for reachability. Use SCC.
Topo Sort on cyclic graphOnly works on DAGs. Run SCC first to collapse cycles.
Ford-Fulkerson with non-integer capacitiesMay not terminate. Use Edmonds-Karp instead.
Modifying a black box internallyNever modify algorithm internals. Only modify input/output.
Writing pseudocode on the examSerious point deduction. Write English narrative only.
Forgetting to convert Kruskal's edges[] to adj listCan't run BFS/DFS on an edge list. Convert first, O(n).
Adding constant to all edges to fix negative weightsDoes NOT work. Changes which path is shortest.
dist[v][v] can be positiveIt's always 0 (no neg cycle) or negative (neg cycle). Never positive.
